Capítulo 1

Ámbito del Proyecto e Introducción

El presente proyecto, denominado HIDROSYS EC., consiste en el análisis, diseño e implementación de un Sistema de Gestión Integral para la automatización de agendamientos de visitas técnicas de instalación y mantenimiento de sistemas hidráulicos, agua potable y gas. La empresa opera en la provincia del Cañar, atendiendo cantones clave como Azogues, Cañar y Biblián.

El sistema destaca por integrar tres componentes fundamentales: un frontend interactivo con arquitectura de micro-animaciones y diseño glassmorphism, un backend robusto conectado a base de datos PostgreSQL, y un asistente automatizado de WhatsApp (WhatsApp Bot) que permite a los usuarios agendar visitas mediante dictado de voz y procesamiento de lenguaje natural o comandos rápidos estructurados.

Requerimientos

1. Necesidades del Usuario

Tras efectuar el levantamiento de información y entrevistas estructuradas a los stakeholders de HIDROSYS EC., se han detectado los siguientes inconvenientes críticos en el flujo tradicional de la empresa:

  1. Falta de un Canal Automatizado de Agendamiento: Los clientes debían llamar o enviar mensajes manuales por WhatsApp a la oficina para agendar una visita técnica. Esto generaba cuellos de botella en la atención y tiempos de espera elevados.
  2. Coordinación Ineficiente de Pagos por Transferencia: El proceso de validación de comprobantes de pago se realizaba de manera dispersa. La secretaria debía verificar la cuenta bancaria manualmente y notificar por teléfono la aprobación del pago al cliente.
  3. Falta de Seguimiento en Tiempo Real: El administrador y los técnicos carecían de un panel consolidado de visitas agendadas, lo que dificultaba la asignación del personal técnico según la parroquia/zona del cliente.
  4. Ausencia de Registro de Satisfacción Post-Servicio: La retroalimentación de la calidad del servicio técnico no era registrada formalmente, impidiendo medir el nivel de satisfacción al cliente.

Interacciones y Flujo

4. Casos de Uso de Contexto

El contexto de HIDROSYS EC. se basa en el flujo del servicio técnico: El Cliente genera el agendamiento y reporta el pago (por web o por WhatsApp). El Administrador valida la transacción bancaria y aprueba la cita. El Asistente Virtual (Bot) notifica inmediatamente la aprobación al Cliente y solicita la confirmación de su disponibilidad. El Técnico completa el trabajo y sube el reporte final.

Especificación de Casos de Uso

5. Casos de Uso Detallado

A continuación se detalla la especificación del Caso de Uso clave: Aprobación de Citas y Notificación Automática.

Caso de Uso: CUD-03: Aprobación de Citas y Asignación Técnica con Notificación WhatsApp
Actores: Administrador (Iniciador), Cliente (Receptor), Técnico.
Precondiciones: El cliente debe tener una cita registrada como "Reportado" con comprobante de pago subido. El bot de WhatsApp debe estar "Conectado".
Flujo Principal (Happy Path): 1. El Administrador inicia sesión en el panel web.
2. Navega a "Agenda y Citas" y selecciona una cita reciente en estado "Reportado".
3. Revisa la imagen del comprobante de transferencia bancaria.
4. Asigna un Técnico disponible y presiona "Aprobar Pago".
5. El servidor procesa la solicitud, actualiza la cita a "Confirmado".
6. El bot de WhatsApp envía automáticamente un mensaje al JID del cliente notificándole que su pago ha sido aprobado e incluye la fecha y nombre del técnico asignado, solicitando su confirmación final.
Flujos Alternativos: - Pago Inválido: El administrador presiona "Rechazar Pago". La cita cambia a "Pre-agendado" y el bot notifica al cliente que debe subir un comprobante válido.
- Bot Desconectado: Si el bot pierde conexión, el sistema almacena la aprobación en PostgreSQL y muestra una alerta en el panel administrador solicitando reiniciar conexión desde la pestaña QR.

Escenarios de Interacción

6. Casos de Uso de Escenario

Escenario de Confirmación del Cliente: El cliente recibe la aprobación de pago en su WhatsApp. El cliente presiona o escribe "SÍ". El bot procesa el mensaje, busca la cita activa en la base de datos de Hidrosys, la cambia al estado "Confirmado por Cliente" y responde con un mensaje de agradecimiento. Si el cliente escribe "NO", el bot cancela el flujo y le indica que se comunique con soporte telefónico.

Diseño y Experiencia de Usuario

9. El Mockup

El diseño del sistema web de HIDROSYS EC. rompe con las plantillas genéricas. Se ha estructurado bajo un concepto visual moderno que incluye:

  • Estilo Glassmorphism: Tarjetas con fondos blancos semi-transparentes (`rgba(255,255,255,0.85)`), desenfoque de fondo (`backdrop-filter: blur(10px)`) y bordes muy finos.
  • Paleta de Colores Curada: Azul pizarra profundo para fondos laterales (`#0f172a`), verde esmeralda para éxitos y confirmaciones (`#10b981`), y azul eléctrico para botones principales (`#2563eb`).
  • Tipografía Moderna: Google Fonts Outfit para encabezados prominentes y limpios, combinada con Inter para textos de lectura fluida.
  • Barra Lateral Responsiva: Transición suave para expandir y colapsar el menú en dispositivos móviles y computadoras.

Arquitectura Tecnológica

10. Prototipo del Sistema

El prototipo de HIDROSYS EC. está desarrollado como una aplicación web real SPA (Single Page Application) e integra las siguientes tecnologías:

Arquitectura del Software

  • Servidor Backend: Node.js con Express, que expone APIs REST seguras protegidas por tokens de sesión.
  • Base de Datos: PostgreSQL, almacenando tablas relacionales para `appointments`, `clients`, `technicians`, y `surveys` con migraciones automáticas.
  • Frontend: JavaScript vainilla estructurado modularmente con selectores CSS y peticiones HTTP (`fetch`).
  • Integración WhatsApp (Baileys): Biblioteca Node.js que emula un cliente de WhatsApp Web, conectándose a través del escaneo del código QR generado en vivo y manejando eventos de mensajes y audios.
