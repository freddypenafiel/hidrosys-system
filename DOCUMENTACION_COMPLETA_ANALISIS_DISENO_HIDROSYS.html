Capítulo 1

Ámbito del Software

¿De qué trata el proyecto?

Este proyecto consiste en el desarrollo de un Sistema de Gestión Integral de visitas e inspecciones técnicas para la empresa HIDROSYS EC., especializada en la instalación y mantenimiento de sistemas hidráulicos, agua potable y redes de gas. El objetivo es automatizar y coordinar en tiempo real los agendamientos, validaciones de pagos y asignación de técnicos, optimizando los canales de comunicación mediante la vinculación directa de un bot de WhatsApp con la base de datos PostgreSQL de la empresa.

Problemas a resolver

Actualmente, la gestión de agendamientos en Hidrosys presentaba serios problemas logísticos:

  • Falta de un Canal Automatizado de Agendamiento: Los clientes debían realizar llamadas telefónicas reiteradas o coordinar por chat manual de WhatsApp con la oficina central, lo que generaba pérdida de tiempo.
  • Proceso de Validación de Pagos Manual: La secretaria verificaba las transferencias bancarias manualmente contra el estado de cuenta y luego llamaba para confirmar la aprobación, retrasando la asignación del técnico.
  • Falta de Seguimiento en Tiempo Real: Ausencia de un panel para el administrador que ordene las citas recientes primero, perdiendo la trazabilidad de las solicitudes urgentes.
  • Pérdida de Conexión del Bot: La imposibilidad de vincular o restablecer de manera autónoma el canal de WhatsApp obligaba a depender constantemente de un programador especializado.

Capítulo 2

Factibilidad del Proyecto

Factibilidad Económica

El costo de horas hombre de desarrollo del sistema se estima en un costo nominal de $5.00 dólares la hora. Estimando un tiempo aproximado de 2,000 horas de trabajo para completar las fases de análisis, modelado base, implementación del backend Node/PostgreSQL, configuración de Baileys y diseño front-end, el costo comercial del proyecto asciende a $10,000.00 dólares. Dado que el proyecto es realizado de forma individual por el estudiante Freddy Peñafiel bajo la tutoría del Ing. Fabricio Lucero, la empresa Hidrosys no asume costos de desarrollo, limitándose únicamente a los gastos de hosting y dominio web estimados en $11.99 anuales.

Factibilidad Técnica

El sistema de Hidrosys se apoya en tecnologías consolidadas de alto rendimiento. En el backend se utiliza Node.js con Express, PostgreSQL como motor de base de datos relacional y Baileys para la emulación del protocolo de WhatsApp Web en el servidor local. En el frontend se implementa HTML5, CSS3 vainilla (sin frameworks pesados) y Javascript nativo para mayor velocidad.

Capítulo 7

Mockup y Prototipo del Sistema

El prototipo de HIDROSYS EC. está completamente desarrollado utilizando una arquitectura de comunicación en tiempo real. La base de datos relacional PostgreSQL contiene esquemas estructurados para resguardar la consistencia transaccional:

  • Tabla `appointments`: Almacena ID único, datos de contacto del cliente, dirección georreferenciada en Cañar, observaciones transcritas por voz, ruta del comprobante de transferencia y estado logístico (Pre-agendado, Reportado, Confirmado, Confirmado por Cliente).
  • Tabla `technicians`: Almacena ID, nombre del técnico de campo, especialidad y zonas geográficas asignadas.
  • Tabla `clients`: Resguarda historial de contactos de clientes para agilizar agendamientos recurrentes.
  • Tabla `surveys`: Almacena las encuestas de satisfacción (estrellas y comentarios) que los clientes envían tras completarse la visita.
