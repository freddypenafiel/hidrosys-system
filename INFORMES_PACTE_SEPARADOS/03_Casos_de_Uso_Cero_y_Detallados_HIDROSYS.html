Nombre: CUD-01: Sincronización QR en Tiempo Real desde el Panel Web
Descripción: Este caso de uso describe el proceso mediante el cual el Administrador Principal o Dueño de Hidrosys EC. escanea el código QR desde su panel web para conectar en tiempo real el teléfono corporativo con el servidor de WhatsApp Bot.
Actores: Dueño / Administrador Principal, Servidor Node.js (Baileys WebSocket)
Precondiciones: El administrador debe haber iniciado sesión como Super Admin y contar con sesión válida.
Flujo Normal: 1. El administrador accede al panel web en localhost:3000.
2. Hace clic en la pestaña lateral 📱 Escanear QR / WhatsApp.
3. El sistema consulta en vivo al endpoint /api/wa/status.
4. Si el bot está desconectado, se renderiza el código QR en pantalla.
5. El administrador apunta la cámara de WhatsApp de su celular y escanea el código.
6. El servidor confirma la autenticación y actualiza el estado a '¡Dispositivo Conectado! ✅'.
Flujo alternativo: A1. Si el código QR expira antes del escaneo, el administrador hace clic en '🔄 Reiniciar Conexión / Generar Nuevo QR' para obtener un QR nuevo al instante.
A2. Si el dispositivo ya estaba conectado, el panel muestra directamente la tarjeta verde de conexión.
Postcondiciones: El bot de WhatsApp queda conectado, sincronizado y activo para responder a clientes.
Reglas de Negocio: Solo usuarios con rol Administrador autenticados pueden visualizar y reiniciar el servicio QR.
Requerimientos No Funcionales: El QR debe renderizarse en menos de 2 segundos. La sincronización se refleja automáticamente sin recargar la página.
Comentarios o Enlaces: Relacionado con HU-03 y con el módulo de conexión Baileys en server.js.

Nombre: CUD-02: Agendamiento Automático de Citas de Agua y Gas por WhatsApp
Descripción: Describe el proceso conversacional por el cual un ciudadano agenda una inspección técnica interactuando con el bot de WhatsApp utilizando el menú numérico estructurado.
Actores: Cliente / Ciudadano, Asistente Virtual WhatsApp Bot, Base de Datos Hidrosys
Precondiciones: El bot debe encontrarse conectado al número corporativo (+593 968245633).
Flujo Normal: 1. El cliente envía un mensaje inicial al número corporativo.
2. El bot responde en <1.5s mostrando las opciones numeradas (1, 2, 3).
3. El cliente responde con '1' (Agendar Cita).
4. El bot solicita paso a paso: Nombre, Dirección Exacta, Tipo de Servicio y Fecha.
5. El servidor inserta la cita en la tabla de base de datos con estado 'Pendiente'.
6. El bot envía al cliente la confirmación oficial con el resumen de la orden.
Flujo alternativo: A1. Si el cliente escribe un texto no numérico en el menú inicial, el bot recuerda amablemente seleccionar una opción (1, 2 o 3).
Postcondiciones: La orden de trabajo queda almacenada formalmente y visible en el panel web del operador.
Reglas de Negocio: El menú debe basarse estrictamente en números simples para garantizar compatibilidad del 100% en todos los celulares.
Requerimientos No Funcionales: El tiempo de respuesta entre mensajes conversacionales no debe exceder los 1.5 segundos.
Comentarios o Enlaces: Relacionado con HU-01, HU-02 y el archivo whatsapp/flows.js.
