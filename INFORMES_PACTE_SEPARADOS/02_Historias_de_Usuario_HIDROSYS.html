ID: HU-01 Título: Menú Numérico en WhatsApp 100% Confiable
Descripción: Como cliente o ciudadano que escribe al bot de WhatsApp de Hidrosys EC., quiero recibir un menú estructurado con opciones numeradas simples (1, 2, 3), para seleccionar mi trámite sin sufrir bloqueos ni errores de incompatibilidad en móviles.
Estimación: 3 puntos de historia Prioridad: 100 (Alta) Dependiente de: Arquitectura Base Baileys Node.js
Pruebas de
aceptación:
  1. Al enviar 'Hola' o cualquier saludo, el bot debe responder en menos de 1.5 segundos con las opciones 1, 2 y 3 en texto plano.
  2. Si el usuario responde '1', el bot inicia el agendamiento de cita técnica.
  3. Si el usuario responde '2', el bot informa sobre zonas de cobertura.
  4. Tolerancia a entradas inválidas con mensaje orientador amable.

ID: HU-02 Título: Agendamiento Guiado de Citas e Inspecciones Técnicas
Descripción: Como cliente que requiere revisión o mantenimiento de agua y gas, quiero ingresar paso a paso mi Nombre, Dirección, Servicio y Fecha por WhatsApp, para dejar registrada mi cita formal sin llamadas telefónicas.
Estimación: 5 puntos de historia Prioridad: 100 (Alta) Dependiente de: HU-01 (Menú Principal)
Pruebas de
aceptación:
  1. El bot solicita de forma ordenada: Nombre Completo, Dirección Exacta, Tipo de Servicio y Fecha.
  2. Los datos ingresados se insertan en la tabla transaccional de base de datos con estado 'Pendiente'.
  3. El bot emite un comprobante formal confirmando el agendamiento exitoso.

ID: HU-03 Título: Reconexión Autónoma mediante Pestaña 'Escanear QR / WhatsApp'
Descripción: Como Administrador Principal o Dueño de Hidrosys EC., quiero disponer de la pestaña 'Escanear QR / WhatsApp' en el panel web que muestre el QR en tiempo real, para conectar el celular corporativo al instante ante cualquier pérdida de sesión sin requerir programadores.
Estimación: 8 puntos de historia Prioridad: 100 (Alta) Dependiente de: Backend Express /api/wa/status
Pruebas de
aceptación:
  1. El menú lateral de localhost:3000 incluye la opción '📱 Escanear QR / WhatsApp'.
  2. Si el bot está conectado, la interfaz muestra '¡Dispositivo Conectado! ✅'.
  3. Si la sesión se desconecta, se dibuja en pantalla el código QR listo para escanear.
  4. Incluye botón protegido para reiniciar y generar un nuevo QR al instante.

ID: HU-04 Título: Seguridad Institucional mediante Middleware requireAuth
Descripción: Como Administrador del Sistema, quiero que las peticiones administrativas a la API REST verifiquen la presencia de un token de sesión, para evitar que usuarios no autorizados manipulen citas o reinicien el bot.
Estimación: 5 puntos de historia Prioridad: 100 (Alta) Dependiente de: Módulo de Sesiones Express
Pruebas de
aceptación:
  1. El middleware requireAuth inspecciona la cabecera x-session-token en endpoints sensibles.
  2. Si el token es inválido o faltante, el servidor rechaza con HTTP 401 Unauthorized.
